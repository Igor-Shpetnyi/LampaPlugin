# UAFLIX — плагін для Lampa

Плагін для особистого використання, який шукає фільми та серіали на [uafix.net](https://uafix.net/)
і відтворює їх у нативному плеєрі Lampa. Відео резолвиться через бекенд-плеєр zetvideo.net.

## Можливості

- Пошук і автоматичне співставлення картки TMDB з відповідною сторінкою на uafix.net
  (за назвою, роком і типом — фільм/серіал). Якщо однозначно визначити не вдалось —
  показується список кандидатів на вибір.
- Окрема кнопка **UAFLIX** на сторінці фільму/серіалу (не замінює вбудовану «Дивитись онлайн»).
- Фільми та серіали з повним архівом серій на uafix.net (`/serial/{id}`).
- Серіали «в процесі виходу» (`/vod/{id}`, по одній серії за раз):
  - вибір сезону, якщо їх декілька;
  - реальні назви серій (там, де вони є на сайті), а не «Серія N»;
  - постери серій у списку вибору.
- Збереження позиції перегляду (продовження з того місця, де зупинились) —
  окремо для кожної серії.
- Обхід гео-блокування контенту, доступного лише з України (детально нижче).

## Архітектура

Три частини, кожна вирішує окрему проблему:

1. **`uaflix.js`** — сам плагін Lampa. Парсить сторінки uafix.net, підбирає відповідність
   до картки TMDB, резолвить посилання на відео через zetvideo.net і запускає нативний плеєр.
2. **`cors-proxy-worker.js`** (Cloudflare Worker) — uafix.net і zetvideo.net не віддають
   CORS-заголовків, тож прямі запити з Lampa браузером блокуються. Цей Worker проксіює
   запити й переписує URL всередині HLS (`.m3u8`) плейлистів, щоб сегменти відео теж
   йшли через проксі. Він же віддає й сам файл плагіна (маршрут `/uaflix.js`, див.
   нижче «Чому не jsDelivr»).
3. **`api/ua-proxy.js`** (Vercel serverless function) — частина контенту (серіали, що
   зараз виходять) доступна лише з українських IP. Ця функція проксіює запит через
   випадковий безкоштовний публічний SOCKS5-проксі з України. Плагін звертається сюди
   лише тоді, коли швидкий CORS-проксі отримав відмову за геолокацією.

## Встановлення на своєму пристрої

1. Відкрий Lampa.
2. Натисни іконку **шестерні** (Налаштування) у верхньому меню.
3. Обери **«Розширення»**.
4. Натисни **«Додати плагін»** → підтверди попередження **«Встановити плагін»**.
5. У полі «Введіть адресу плагіна» встав:

   ```
   https://uaflix-cors-proxy.igor-shpetnyi.workers.dev/uaflix.js
   ```

6. Натисни **«Готово»**, потім повністю перезапусти Lampa.

Після цього на сторінці фільму/серіалу поруч із «Дивитись» з'явиться кнопка **UAFLIX**.

### Чому не пряме посилання на GitHub / jsDelivr

- `raw.githubusercontent.com` — Lampa успішно проходить перевірку при встановленні
  (показує «200 Робочий»), але на реальному старті застосунку плагін **не завантажується**
  («не вдалося завантажити») — це особливість власного кешу плагінів Lampa, перевірено
  наживо.
- `cdn.jsdelivr.net/gh/...@main/...` — jsDelivr тримає власне дзеркало репозиторію і
  синхронізує його з затримкою; після пуша нового коміту файл там може ще довго
  лишатись застарілим, а примусовий purge кешу цього не виправляє (дзеркало застаріле,
  не лише крайовий кеш).

Тому плагін віддається через маршрут `/uaflix.js` на вже наявному Cloudflare Worker —
він тягне файл з GitHub наживо на кожен запит, без жодного кешування. Посилання
достатньо додати один раз — після майбутніх оновлень коду міняти нічого не треба.

## Налаштування плагіна

У Налаштуваннях Lampa з'являється розділ **UAFLIX** з полями:

- **Домен** — за замовчуванням `uafix.net`, міняти не потрібно.
- **CORS-проксі** — адреса Cloudflare Worker. Щоб вимкнути (прямі запити без проксі),
  введи `-`.
- **UA-проксі** — адреса Vercel-функції для гео-заблокованого контенту. Також вимикається
  через `-`.
- **Очистити кеш** — скидає кеш співставлень фільм↔сторінка uafix.net та кеш дерева серій.

Значення за замовчуванням робочі, змінювати їх без потреби не треба.

## Структура репозиторію

| Файл | Призначення |
|---|---|
| `uaflix.js` | сам плагін Lampa |
| `cors-proxy-worker.js` | код Cloudflare Worker (CORS-проксі + роздача плагіна) |
| `worker-metadata.json` | метадані для ручного редеплою Worker через Cloudflare API |
| `api/ua-proxy.js` | Vercel serverless function (обхід гео-блокування) |
| `package.json` | залежність `socks-proxy-agent` для Vercel-функції |

Worker і Vercel-функція деплояться незалежно від плагіна (Worker — вручну через
Cloudflare API з `worker-metadata.json`, Vercel-функція — автоматично при пуші в
`main` через GitHub-інтеграцію). Файл `uaflix.js` самостійного деплою не потребує —
Worker завжди роздає його актуальну версію напряму з GitHub.

## Відомі особливості

- uafix.net у себе в навігації іноді показує вкладки на сезони, яких по факту ще немає
  (запланований TMDB сезон без завантажених серій). Якщо обрати такий сезон, плагін
  коректно покаже повідомлення «серії не знайдено» — це не помилка плагіна, а порожній
  розділ на самому сайті.
- Плагін розрахований на особисте використання одного акаунта на uafix.net-контенті;
  сайт і безкоштовні публічні проксі можуть змінюватись або ставати недоступними поза
  контролем цього репозиторію.
